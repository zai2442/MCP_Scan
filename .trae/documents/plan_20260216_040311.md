I have analyzed the current codebase and the requirements for implementing persistent storage of job UUIDs and results.

Here is the detailed implementation plan:

## 1. Dependency Management
- **Action**: Add `mysql-connector-python>=8.0.0` to `requirements.txt`.
- **Reason**: To enable Python applications to connect to the MySQL database.

## 2. Configuration Update
- **Action**: Modify `src/mcp_scan/config.py`.
- **Details**:
    - Add `DatabaseConfig` class with fields: `host`, `port`, `user`, `password`, `database`.
    - Add `database` field to `MCPConfig` with default values matching the `docker-compose.yml` setup (localhost, 3306, root, root, job_result_db).

## 3. Database Schema Enhancement
- **Action**: Update database schema to include a `status` column.
- **Details**:
    - Update `docs/mvp-p0/job-result-system/init/01-schema.sql` to include `status VARCHAR(20) DEFAULT 'pending'`.
    - Create `docs/mvp-p0/job-result-system/init/03-add-status.sql` for existing deployments.

## 4. Core Database Implementation
- **Action**: Create `src/mcp_scan/core/db.py`.
- **Details**: Implement `DatabaseManager` class to handle:
    - Connection pooling.
    - `save_job(job)`: Serialize `Job` object to JSON and upsert into `job_results` table.
    - `get_job(job_id)`: Fetch job data and deserialize back to `Job` object.
    - `update_status(job_id, status)`: Efficiently update job status.

## 5. Scheduler Integration
- **Action**: Modify `src/mcp_scan/core/scheduler.py`.
- **Details**:
    - Initialize `DatabaseManager`.
    - `create_job`: Save initial job state to DB.
    - `run_job`: Update status to `RUNNING` in DB at start, and `COMPLETED`/`FAILED` at end.
    - `_process_task_result`: Save updated job state to DB.
    - `get_job`: Attempt to fetch from DB if not found in memory (enabling persistence across CLI runs).

## 6. CLI Enhancement
- **Action**: Modify `src/mcp_scan/cli.py`.
- **Details**:
    - Update `start` command to ensure job persistence.
    - Update `status` command to fetch status from DB via `scheduler.get_job`.
    - Implement `report` command with `-o` support:
        - Fetch job from DB using `scheduler.get_job(job_id)`.
        - Export full job data (including results) to the specified JSON file.
        - Handle errors (job not found, file write permission).

## 7. Verification
- **Action**: Create a test script `tests/test_db_persistence.py` to verify:
    - Job creation and retrieval from DB.
    - Status updates.
    - JSON serialization/deserialization correctness.
